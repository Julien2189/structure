📐 3.1 — Les Tenseurs

Un tenseur est la généralisation mathématique du concept de tableau à n dimensions. C'est la brique atomique du machine learning : toutes les données, tous les poids d'un réseau, toutes les activations sont représentés sous forme de tenseurs.

Pourquoi des tenseurs ?

Le choix du tenseur n'est pas arbitraire. Il reflète directement la structure mathématique des données :

  • Un seul nombre (température, score) → scalaire de rang 0
  • Un vecteur de features (taille, poids, âge) → rang 1
  • Un lot d'exemples (N lignes × M features) → matrice de rang 2
  • Une image RGB (hauteur × largeur × 3 canaux) → rang 3
  • Un lot d'images (batch × hauteur × largeur × canaux) → rang 4
💡 Intuition géométrique

Imagine qu'un scalaire est un point. Un vecteur est une ligne. Une matrice est une feuille de papier. Un tenseur 3D est un cube. La shape dit combien de cases il y a dans chaque direction.

Les opérations fondamentales

Toutes les opérations du deep learning se résument à quelques opérations algébriques de base sur les tenseurs :

Multiplication Matricielle — le cœur d'une couche Dense
Y = X · W + b
X = tenseur d'entrée [batch, in_features]
W = matrice de poids [in_features, out_features]
b = vecteur de biais [out_features]
Y = tenseur de sortie [batch, out_features]

⚠️ Compatibilité de shapes

En algèbre linéaire, on ne peut multiplier A · B que si le nombre de colonnes de A = nombre de lignes de B. Exemple : [32, 64] · [64, 10] → valide, donne [32, 10]. Si les shapes sont incompatibles, TensorFlow lève une erreur immédiate.

📦 3.2 — Shape, dtype et Batch

Trois attributs à maîtriser absolument. La plupart des erreurs TensorFlow viennent d'une confusion sur ces trois notions.

La Shape — lire et interpréter

La shape est une liste d'entiers qui décrit les dimensions d'un tenseur. Se lit de gauche (extérieur) à droite (intérieur).

[]
Scalaire — un seul nombre
tf.scalar(3.14)
[4]
Vecteur de 4 éléments
tf.tensor1d([1,2,3,4])
[32, 10]
Batch de 32 exemples, 10 features chacun
entrée Dense classique
[28, 28, 1]
Image en niveaux de gris 28×28
MNIST un seul sample
[64, 28, 28, 3]
Batch de 64 images RGB 28×28
batch_size × H × W × canaux
[32, 50, 1]
Batch 32, séquence de 50 pas, 1 feature
batch × timesteps × features (LSTM)

Le dtype — précision numérique

Le type de données détermine la précision et la plage de valeurs possibles. Un mauvais dtype peut silencieusement produire des résultats faux.

dtype Bits Cas d'usage Précision
float32 32 Par défaut pour les poids et activations. Standard en ML. ~7 chiffres significatifs
float16 16 GPU modernes, mixed precision training. 2× plus rapide. ~3 chiffres — risque d'overflow
float64 64 Calculs scientifiques de haute précision. Rarement utile en DL. ~15 chiffres significatifs
int32 32 Indices, labels, compteurs. Pas pour les poids. Entier exact ±2.1 milliards
bool 8 Masques, conditions, filtres sur tenseurs. True / False uniquement

Le Batch — traiter en paquets

Au lieu de traiter les exemples un par un, on les regroupe en batchs. Ce n'est pas qu'une optimisation : c'est mathématiquement fondamental pour l'estimation des gradients.

Descente de gradient stochastique — estimation du gradient
∇L ≈ (1/B) · Σᵢ ∇Lᵢ(θ)
B = taille du batch (batch size)
∇L = gradient moyen de la loss sur le batch
∇Lᵢ = gradient de la loss sur l'exemple i
θ = paramètres du modèle (poids + biais)

Plus le batch est grand, plus l'estimation du gradient est précise (proche du vrai gradient), mais plus on consomme de mémoire GPU. Le compromis standard :

  • Batch size 32 — bon équilibre bruit/vitesse pour la plupart des cas
  • Batch size 64-256 — entraînements plus stables, GPU large
  • Batch size 1 — SGD pur, très bruité mais parfois utile en online learning
💡 Règle du pouce

Si le modèle ne converge pas, essaie de réduire le batch size. Un gradient plus bruité peut aider à sortir des minima locaux. Si l'entraînement est trop lent et la mémoire GPU disponible, augmente le batch size.

🔗 3.3 — Le Graphe de Calcul & Eager Execution

TensorFlow construit une représentation interne de tous les calculs avant de les exécuter. Cette représentation est un graphe orienté acyclique (DAG).

Qu'est-ce qu'un graphe de calcul ?

Chaque opération mathématique devient un nœud du graphe. Les tenseurs qui circulent d'une opération à l'autre sont les arêtes. Cette structure permet à TensorFlow de :

  • Optimiser automatiquement — réordonner les opérations pour maximiser le parallélisme GPU
  • Calculer les gradients — en "remontant" le graphe dans le sens inverse (c'est la backpropagation)
  • Compiler et déployer — exporter le graphe figé pour la production
Exemple de graphe — calcul de y = relu(x·W + b)
x → [matMul(W)] → z₁ → [add(b)] → z₂ → [relu] → y
Chaque → représente un tenseur qui circule.
Chaque [opération] est un nœud du graphe.
En backward : le gradient remonte de y vers x, en passant par chaque nœud.

Eager vs Graph — le duel

Eager Execution (TF2 par défaut)

Les opérations s'exécutent immédiatement, ligne par ligne, comme du code normal. Tu peux inspecter les valeurs à n'importe quel moment avec .print() ou print().

  • ✅ Debugging facile
  • ✅ Comportement intuitif
  • ⚠️ Moins optimisé pour la production
@tf.function — Mode Graphe

Le décorateur @tf.function convertit une fonction Python en graphe TF optimisé. TF trace l'exécution une fois, compile, puis réutilise le graphe compilé.

  • ✅ 2× à 10× plus rapide en production
  • ✅ Exportable (SavedModel)
  • ⚠️ Debugging plus complexe
TensorFlow.js
Eager (défaut en TF.js)JS
// Eager : tout s'exécute immédiatement
const x = tf.tensor2d([[1,2],[3,4]]);
const W = tf.tensor2d([[0.5,0],[0,0.5]]);
const b = tf.tensor1d([1,1]);

const z  = x.matMul(W);      // calculé ici
z.print();                    // tu peux l'inspecter

const z2 = z.add(b);          // calculé ici
const y  = z2.relu();         // calculé ici
y.print();                    // résultat final
TensorFlow Python
@tf.function — graphe compiléPython
import tensorflow as tf

# Eager (par défaut — TF2)
x = tf.constant([[1.,2.]])
W = tf.constant([[0.5],[0.5]])
b = tf.constant([1.])
z = tf.matmul(x, W) + b
print(z)  # tf.Tensor([[2.5]], ...)

# Mode graphe avec @tf.function
@tf.function
def forward(x, W, b):
  return tf.nn.relu(tf.matmul(x, W) + b)

# 1ère appel : TF trace et compile le graphe
# Appels suivants : graphe réutilisé → RAPIDE
result = forward(x, W, b)
💡 Quand utiliser @tf.function ?

Pendant le développement : laisse Eager pour débugger facilement. En production ou pour accélérer l'entraînement de longs runs : enveloppe la boucle d'entraînement dans @tf.function. Le gain est souvent de 2× à 5× sur GPU.

⚡ 3.4 — La Rétropropagation (Backpropagation)

La backpropagation est l'algorithme qui permet au réseau d'apprendre. C'est une application élégante de la règle de la chaîne des dérivées, appliquée au graphe de calcul pour calculer efficacement le gradient de chaque paramètre.

Vue d'ensemble — le cycle d'entraînement

Étape 1
Forward Pass
Les données traversent le réseau couche par couche. On calcule les activations à chaque niveau.
ŷ = f(x ; θ)
Étape 2
Calcul de la Loss
On mesure l'écart entre la prédiction et la vraie valeur via une fonction de coût.
L = loss(ŷ, y)
Étape 3
Backward Pass
On remonte le graphe en calculant le gradient de L par rapport à chaque paramètre.
∂L/∂θ = ?
Étape 4
Mise à jour
L'optimiseur ajuste les poids dans la direction opposée au gradient.
θ ← θ - α·∇θL

La Règle de la Chaîne — le moteur de la backprop

Si y = f(g(x)), alors la dérivée de y par rapport à x est :

Règle de la chaîne (Chain Rule)
dy/dx = (dy/dg) × (dg/dx)
La dérivée d'une fonction composée = produit des dérivées de chaque couche.
En deep learning : on "multiplie les gradients" en remontant couche par couche.

En pratique, pour un réseau à 3 couches :

Gradient sur les poids de la couche 1
∂L/∂W₁ = (∂L/∂a₃) · (∂a₃/∂a₂) · (∂a₂/∂a₁) · (∂a₁/∂W₁)
a₁, a₂, a₃ = activations des couches 1, 2, 3
W₁ = poids de la couche 1
Chaque facteur correspond à un nœud du graphe parcouru à rebours.

Le Forward Pass — formule de couche Dense

À chaque couche Dense, le calcul est :

Sortie d'une couche — Forward
z = W · x + b   →   a = σ(z)
x = entrée de la couche [in_features]
W = matrice de poids [out_features, in_features]
b = vecteur de biais [out_features]
z = pré-activation (linear combination)
a = activation — sortie de la couche
σ = fonction d'activation (ReLU, sigmoid, etc.)

Les Fonctions de Loss — mesurer l'erreur

Loss Formule Quand l'utiliser
MSE
Mean Squared Error		 (1/n) Σ (ŷᵢ - yᵢ)² Régression — prédire un nombre
MAE
Mean Absolute Error		 (1/n) Σ |ŷᵢ - yᵢ| Régression robuste aux outliers
Binary CE
Cross-Entropy binaire		 -[y·log(ŷ) + (1-y)·log(1-ŷ)] Classification binaire (oui/non)
Categorical CE
Cross-Entropy catégorielle		 -Σ yᵢ·log(ŷᵢ) Classification multi-classes (N > 2)

Implémenter manuellement — comprendre GradientTape

TensorFlow.js
Gradient manuel avec tf.gradJS
// Calculer le gradient de f(x) = x²
const f = x => x.square();
const df = tf.grad(f);  // df/dx = 2x

const x = tf.tensor1d([3]);
df(x).print();             // [6] — car d(x²)/dx = 2x = 2×3

// Gradient d'une loss MSE
const mse = (pred, label) =>
  pred.sub(label).square().mean();

const pred  = tf.tensor1d([0.8, 0.2]);
const label = tf.tensor1d([1.0, 0.0]);

const dLoss = tf.grad(p => mse(p, label));
dLoss(pred).print(); // gradient par rapport aux preds
TensorFlow Python
Gradient manuel avec GradientTapePython
import tensorflow as tf

# GradientTape : enregistre les opérations
# pour calculer les gradients ensuite
W = tf.Variable([[0.5], [0.3]])
b = tf.Variable([0.1])
x = tf.constant([[1., 2.]])
y = tf.constant([[1.]])

with tf.GradientTape() as tape:
  pred = tf.matmul(x, W) + b  # forward
  loss = tf.reduce_mean(
    (pred - y)**2              # MSE
  )

# La tape "rejoue" le graphe à rebours
grads = tape.gradient(loss, [W, b])
# grads[0] = ∂loss/∂W
# grads[1] = ∂loss/∂b

# Appliquer les gradients manuellement
lr = 0.01
W.assign_sub(lr * grads[0])
b.assign_sub(lr * grads[1])
⚠️ Vanishing & Exploding Gradients

Dans un réseau profond, les gradients se multiplient couche par couche (règle de la chaîne). Si chaque gradient est < 1, le produit tend vers 0 → vanishing gradient (les premières couches n'apprennent plus). Si chaque gradient > 1, le produit explose → exploding gradient. Solutions : ReLU (atténue le vanishing), Batch Norm, gradient clipping, architectures Residual (ResNet).

🎯 3.5 — Les Optimiseurs

Une fois le gradient calculé, l'optimiseur décide comment ajuster les poids. Ce n'est pas trivial : le gradient seul dit la direction, mais pas la taille du pas ni comment s'adapter au terrain.

Le problème — la surface de loss

Imagine la loss function comme un relief montagneux. Le gradient pointe vers la pente montante. On veut descendre vers la vallée (le minimum). Le problème : la surface est parsemée de minima locaux, de plateaux et de selles.

Mise à jour des poids — formule générale
θₜ₊₁ = θₜ - α · update(∇θL)
θₜ = poids à l'itération t
α = learning rate (taux d'apprentissage)
∇θL = gradient de la loss par rapport aux poids
update() = transformation du gradient selon l'optimiseur

Les 4 grands optimiseurs

SGD
Stochastic Gradient Descent
Le plus simple. Suit le gradient brut, éventuellement avec momentum pour accélérer dans les directions persistantes.
θₜ₊₁ = θₜ - α · ∇L
// Avec momentum :
vₜ₊₁ = β·vₜ + (1-β)·∇L
θₜ₊₁ = θₜ - α·vₜ₊₁
→ Bons résultats finaux, mais convergence lente. Utile pour fine-tuning.
Adam
Adaptive Moment Estimation
Combine momentum (1er moment) et adaptation du learning rate par paramètre (2ème moment). Choix par défaut dans 90% des cas.
m = β₁·m + (1-β₁)·g
v = β₂·v + (1-β₂)·g²
θ = θ - α · m̂ / (√v̂ + ε)
→ Idéal pour débuter. β₁=0.9, β₂=0.999, α=0.001.
RMSprop
Root Mean Square Propagation
Adapte le learning rate par paramètre en divisant par la racine de la moyenne mobile des gradients au carré. Précurseur d'Adam.
v = ρ·v + (1-ρ)·g²
θ = θ - α · g / (√v + ε)
→ Robuste pour certaines séquences et problèmes non stationnaires.
AdamW
Adam + Weight Decay découplé
Correction d'Adam : le weight decay (régularisation L2) est appliqué directement aux poids, pas aux gradients. Meilleure généralisation.
// Adam classique :
θ ← Adam(θ) - α·λ·θ
// AdamW découplé :
θ ← Adam(θ) ; θ ← θ·(1-λ)
→ Recommandé pour les Transformers et gros modèles.

Le Learning Rate — le paramètre le plus important

Le learning rate α contrôle la taille du pas à chaque mise à jour. C'est souvent le paramètre qui a le plus d'impact sur les résultats :

α trop grand

Le modèle "saute" par-dessus les minima. La loss diverge ou oscille sans converger. Symptôme : loss qui monte, NaN dans les poids.

α trop petit

La convergence est correcte mais très lente. Risque de rester bloqué dans un mauvais minimum local. L'entraînement prend des heures sans raison.

Learning Rate Scheduling — réduire α au fil du temps
αₜ = α₀ · decay_rate ^ (t / decay_steps)
α₀ = learning rate initial (ex: 0.001)
t = itération courante
decay_rate = facteur de réduction (ex: 0.96)
Un scheduler réduit α quand la loss stagne → affine la convergence.
TensorFlow.js
Configurer un optimiseurJS
// Adam avec learning rate personnalisé
const opt = tf.train.adam(
  0.001,  // learningRate
  0.9,    // beta1 (momentum)

// SGD avec momentum

// Compiler le modèle avec l'optimiseur

});

// Utiliser directement l'optimiseur
const lossFn = () =>
  tf.tidy(() => mse(model.predict(X), Y));

opt.minimize(lossFn);  // calcule et applique

⚙️ Tableau de décision — quel optimiseur choisir ?

Débuter / usage général → Adam (lr=0.001)
Gros modèle / Transformer → AdamW (weight_decay=0.01)
Fine-tuning d'un modèle pré-entraîné → SGD + momentum (lr très petit : 1e-4)
Si Adam diverge → réduire le lr (0.0001) ou passer à RMSprop
Toujours → ajouter un scheduler ou ReduceLROnPlateau

Le cycle complet en une boucle

Boucle d'entraînement complète — PythonPython
import tensorflow as tf

# Paramètres du modèle (Variables = poids apprenables)
W = tf.Variable(tf.random.normal([2, 1]))
b = tf.Variable(tf.zeros([1]))
opt = tf.keras.optimizers.Adam(0.01)

for epoch in range(100):
  # ── ÉTAPE 1 & 2 : Forward + Loss ────────────────
  with tf.GradientTape() as tape:
    pred = tf.matmul(X_batch, W) + b  # forward
    loss = tf.reduce_mean((pred - Y_batch)**2)  # MSE

  # ── ÉTAPE 3 : Backward (calcul des gradients) ───
  grads = tape.gradient(loss, [W, b])
  # grads = [∂L/∂W, ∂L/∂b]

  # ── ÉTAPE 4 : Mise à jour (optimiseur) ──────────
  opt.apply_gradients(zip(grads, [W, b]))
  # Adam transforme les grads avant d'appliquer
  # θ ← θ - α · Adam(∇L)

  if epoch % 10 == 0:
    print(f"Epoch {epoch}: loss = {loss.numpy():.4f}")
🎯 Les 4 lois à ne jamais oublier

1. Normalise toujours tes données avant de les passer au modèle.
2. Split avant normalisation — les stats du test ne doivent pas contaminer le train.
3. Choisis la bonne loss selon ton problème (MSE régression, CE classification).
4. Surveille val_loss, pas train_loss — la val_loss est le vrai indicateur de performance.
